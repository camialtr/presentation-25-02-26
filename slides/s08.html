Sem HTMX, você cria um endpoint JSON, faz fetch no JS, limpa o select de Cidades e itera criando options. Muito "código cola".

Implementação

Request Automático

O select "pai" (Estado) avisa ao backend que mudou. O backend devolve o HTML do select "filho" (Cidade).

template_form.html
<select name="estado" hx-get="{% url 'cidades' %}" hx-trigger="change" hx-target="#id_cidade" hx-indicator=".loading"> ...options... </select>
Benefício

O servidor renderiza as options corretas usando os forms do Django. O front apenas substitui o pedaço necessário.
